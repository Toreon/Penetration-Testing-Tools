How to Contribute

Adding a New Tool

To add a new tool to the catalogue:

  1. Create a new YAML file in the data/tools/ directory
  2. Name the file using the tool's ID (e.g., my-tool.yml)
  3. Fill in all required fields following the schema below
  4. Submit a pull request with your changes
 YAML Schema

Each tool YAML file should follow this structure:

id: unique-tool-id
name: Tool Name
summary: Short one-paragraph description
description: | (optional)
  Longer description with multiple paragraphs
categories:
  - category1
  - category2
tags:
  - tag1
  - tag2
platforms:
  - linux
  - windows
  - macos
license: License Name
maturity: stable | active | experimental | archived
website: https://example.com (optional)
docs_url: https://docs.example.com (optional)
download_url: https://download.example.com (optional)
github_repo: owner/repo (optional)
stars: 12345 (optional, cached GitHub stars)
use_cases: (optional)
  - Use case 1
  - Use case 2
related_tools: (optional)
  - tool-id-1
  - tool-id-2
added_at: "2024-01-15" (optional, ISO date)
 Required Fields
  • id: URL-safe identifier (lowercase, hyphens allowed)
  • name: Human-readable tool name
  • summary: One-paragraph description
  • categories: List of applicable categories
  • tags: List of relevant tags
  • platforms: List of supported platforms
  • license: License name or identifier
  • maturity: One of: stable, active, experimental, archived
 Updating Existing Tools

To update an existing tool:

  1. Find the tool's YAML file in data/tools/
  2. Make your changes (e.g., update description, add tags, fix links)
  3. Submit a pull request with your improvements

Common updates include: fixing broken links, adding new tags, updating descriptions, correcting license information, or adding related tools.

Proposing New Categories or Tags

If you believe a new category or tag would be useful, please open an issue to discuss it with the maintainers. Categories should represent broad tool types, while tags can be more specific and numerous.

Example Tool Entry

Here's an example of a complete tool entry:

id: example-tool
name: Example Security Tool
summary: A powerful tool for security testing and analysis.
description: |
  This tool provides comprehensive security testing capabilities
  for web applications and networks.
categories:
  - web_app_scanner
  - vulnerability_scanner
tags:
  - security
  - testing
  - oss
platforms:
  - linux
  - windows
license: MIT
maturity: stable
website: https://example.com
docs_url: https://docs.example.com
github_repo: owner/example-tool
stars: 5000
use_cases:
  - Web application scanning
  - Vulnerability detection
related_tools:
  - sqlmap
  - nessus
added_at: "2024-01-15"

For more detailed information, please refer to the CONTRIBUTING.md file in the repository.

Thank you for contributing to the Pentest Tools Catalogue!
